# バリュー・プロポジション・キャンバス

## 顧客プロファイル

### Customer Jobs（顧客のタスク）

#### 店舗運営者（導入企業）
1. 顧客のリピート率を向上させたい
2. 顧客データを収集・分析して施策に活かしたい
3. LINE公式アカウントを活用して顧客との接点を増やしたい
4. 紙のスタンプカードをデジタル化したい
5. 自社ブランドに合ったデザインで提供したい

#### エンドユーザー（消費者）
1. お得に買い物をしたい
2. スタンプカードを忘れずに持ち歩きたい
3. 自分の特典状況を簡単に確認したい

### Pains（痛み・不満）

#### 店舗運営者
1. 「誰が」スタンプを貯めているか個人特定ができない
2. 購入金額に応じた柔軟なスタンプ付与ができない
3. 会員データベースとスタンプ情報が分散管理されている
4. テンプレート限定でブランドの世界観を表現できない
5. スタンプデータを活用した施策が打てない

#### エンドユーザー
1. 紙のスタンプカードを忘れる・なくす
2. スタンプがいくつ貯まっているか分からない
3. 特典の有効期限を忘れて失効してしまう

### Gains（期待する成果）

#### 店舗運営者
1. 顧客一人ひとりの購買行動を把握したい
2. データに基づいた効果的なキャンペーンを実施したい
3. ロイヤル顧客を育成し、LTVを向上させたい
4. 低コスト・短期間で導入したい

#### エンドユーザー
1. いつでもスマホでスタンプを確認できる安心感
2. ランクアップによる達成感と特典の楽しみ
3. 自分に合ったお得情報を受け取りたい

## バリューマップ

### Products & Services（提供するサービス）

1. **LINE UserID連携スタンプカード** - 個人を特定してスタンプを管理
2. **柔軟な付与ロジック** - 金額連動、イベント参加、商品単位など対応
3. **ランク制度** - 顧客育成を目的とした段階的ランクアップ
4. **フルカスタムデザイン** - テンプレート制約なしのブランドカスタマイズ
5. **セグメント配信** - 蓄積データに基づく自動メッセージ配信
6. **分析ダッシュボード** - 顧客行動分析・可視化機能

### Pain Relievers（痛みの解消方法）

| 痛み | 解消方法 |
|------|----------|
| 誰がスタンプを貯めているか分からない | LINE UserIDとスタンプを紐付け、来店日時・回数を正確トラッキング |
| 柔軟な付与ができない | 金額連動、イベント参加、商品単位など様々な条件設定が可能 |
| 会員データと情報が分散 | 一元管理されたデータベースで顧客情報を統合 |
| ブランド世界観を表現できない | フルカスタムデザインで自由にブランディング |
| データを施策に活かせない | セグメント配信機能で「あと1つでゴールド会員」等の喚起が可能 |

### Gain Creators（成果の実現方法）

| 期待する成果 | 実現方法 |
|--------------|----------|
| 顧客の購買行動把握 | ID管理で「匿名客」から「ロイヤル会員A様」へ進化 |
| 効果的なキャンペーン | 分析ダッシュボードでデータドリブンな意思決定 |
| ロイヤル顧客育成 | ランク制度で段階的な顧客育成を実現 |
| 低コスト・短期間導入 | 既存LINE公式アカウントに追加する形で導入可能 |
